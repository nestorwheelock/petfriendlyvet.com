TECHNICAL IMPLEMENTATION NOTES
================================================================================

1. DOUBLE-ENTRY ACCOUNTING:
   - Every transaction creates balanced journal entries
   - Debits must equal credits (validation enforced)
   - Journal entries are immutable (void to reverse)
   - Automatic entries from billing, payments, inventory

2. PERIOD MANAGEMENT:
   - Fiscal year and monthly period tracking
   - Period close locks transactions for that period
   - Prevents backdated entries to closed periods
   - Year-end closing entries automated

3. HTMX INTERACTIONS:
   - Account balance updates on transaction entry
   - Bill payment modal without page reload
   - Real-time reconciliation matching
   - Drill-down into account details

4. ALPINE.JS COMPONENTS:
   - Journal entry line item editor
   - Bank reconciliation matcher
   - Budget period selector
   - Chart of accounts tree view

5. REPORT GENERATION:
   - Financial statements via WeasyPrint (PDF)
   - Excel export via openpyxl
   - Scheduled report delivery via Celery
   - Comparative period analysis

6. BANK RECONCILIATION:
   - Import CSV/OFX bank statements
   - Auto-matching algorithm
   - Manual match for unrecognized items
   - Adjustment entries for fees/interest

7. INTEGRATIONS:
   - S-020 Billing: auto-AR entries from invoices
